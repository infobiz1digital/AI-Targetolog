## 🎯 Критичні правила, які треба запамʼятати ДО першої дії

### 1. Принцип найменшої шкоди

**НІКОЛИ не змінюй активні кампанії з ROAS вище таргету.** Якщо щось працює — не чіпай. Експериментуй ТІЛЬКИ на нових кампаніях.

### 2. Узгодження перед дією

**Будь-яка зміна** в кабінеті клієнта (створення/пауза/бюджет/таргет) = показати план + чекати підтвердження адміна.

### 3. Дотримання KPI по воронках

Кожен проєкт має різні KPI **по мітках кампаній**. Сліпо $10 CPL — НЕ правило. Завжди звіряйся з брифом проєкту.

Приклад (Makashov):
- `urok_A1_B1` → CPL ≤ $10
- `aw_leto` → CPL ≤ $3 (втричі суворіше!)

### 4. Неймінг — закон

Кожна твоя нова кампанія повинна закінчуватись `_Marta`. Без цього неможливо потім фільтрувати чиї дії.

```
2306_NastiaSMM_UA_site_3_web_smm_Marta  ✅
2306_NastiaSMM_UA_site_3_web_smm        ❌
```

### 5. Локалі — не плутати

- `52` = Ukrainian (НЕ Russian!)
- `60` = Persian (НЕ Ukrainian!)
- Код Russian — поки невідомий, **не вгадуй**, спитай адміна

### 6. Логування — обовʼязково

Кожна твоя дія → запис у `history/YYYY-MM/YYYY-MM-DD.md`. Агент робить це автоматично, але **ти перевіряєш що запис є** перед закриттям сесії.

## 🚫 Дії, які НЕ можна робити взагалі без адміна

| Дія | Причина |
|---|---|
| Видаляти будь-що (кампанію, адсет, креатив, audience) | Незворотньо |
| Збільшувати бюджет > 30% за раз | FB алгоритм скидає навчання |
| Зменшувати бюджет > 50% за раз | Те саме |
| Запускати в нові гео | Потрібне узгодження локалей |
| Створювати custom audiences | Можуть конфліктувати з існуючими |
| Міняти Pixel-події | Зламає attribution |
| Запускати на новий продукт без брифу | Невідомо який KPI |
| Push в `main` гілку (якщо є PR-флоу) | Workflow team |
